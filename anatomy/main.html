Anatomy of a blab

Blab is short for web laboratory, like blog is short for web log. A blab is a short(ish) web page, often involving computation in the STEM fields (Science, Technology, Engineering, Mathematics). The computation is done locally, in the browser, via a [http://puzlet.org/doc/import-export/, special brew of CoffeeScript]. Behind the scenes, a blab is set of files managed and served by GitHub (although entirely local operation is fine). The [https://github.com/puzlet, puzlet.js] application processes and presents the files. The important parts of a blab are:

resources.json

A list of resources to be processed by the puzlet.js app. For example:
  • local .html, .css, .js, .json, .coffee files;
  • .js files from elsewhere on the web;
  • .coffee files from elsewhere (subsequently compiled);
  • GitHub hosted files (via GitHub's API).

blab namespace

The $blab variable is used as a global namespace to pass variables between javascript TBD.

index.html

Every blab has an (often simple) index.html to call the puzlet.js app.
